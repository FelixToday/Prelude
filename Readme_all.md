# Prelude 项目结构与使用说明

本目录提供了适用于早期网站指纹（Website Fingerprinting, WFP）预测的深度学习框架（**Prelude**）及其对比基线模型的实现。核心代码主要收拢在 `Prelude_main` 目录下，下文将详细介绍其结构及各项功能的使用方法。

## 1. 目录结构概览

`Prelude_main` 下主要包含以下两个核心目录：

- **Model/**: 存放所有的模型定义文件，包含了基础特征提取网络（Backbone，包括 Prelude 以及各种对比基线模型）和用于预测与控制是否提前分类的动态门控模块（GateNet）。
- **Run/**: 存放核心的运行脚本与配置文件。所有的模型训练、早期流式测试、动态门控执行脚本，以及超参数敏感性分析脚本均在此目录下。
    - **config/**: 项目各基线与主干网络的具体超参数配置文件（以 `.ini` 结尾）。

## 2. 核心功能及使用方式

建议所有命令的执行和配置修改操作均在 `Prelude_main/Run` 目录下进行。您可以直接通过修改预设好的 `.sh` 脚本及其内部遍历变量来执行对应的实验流程。

### 📌 2.1 常规整流特征分类（基线训练与定比例测评）

如果您需要执行标准流程中的网站指纹攻击训练，以及（基于固定加载比例特征的）准确率测试环节，请使用 `bash_main_process.sh`：

```bash
cd Prelude_main/Run
bash bash_main_process.sh
```

- **功能说明**: 该脚本默认会遍历指定的数据集（如 `CW`, `OW` 等）及指定的基线模型集合（如 `AWF`, `DF`, `TikTok`, `VarCNN`, `CountMamba`, `Prelude` 等），顺序完成完整特征的主干网络训练逻辑，随后通过 `test.py` 测试模型在不同固定流量比例（例如 10% 到 100%）下的识别准确率。
- **关联脚本**: `main.py`（主干网络训练核心）和 `test.py`（定比例测试推断）。

### 📌 2.2 动态门控机制执行（早期分类任务核心部分）

针对 **Prelude** 框架最核心的“提前拒绝与输出判定机制”，我们需要训练额外的门控网络，并对其端到端的早期探测能力进行评估。执行此流程请使用 `bash_gate_process.sh`：

```bash
cd Prelude_main/Run
bash bash_gate_process.sh
```

- **功能说明**: 该脚本负责端对端调用门控机制训练与推断。使用者可在脚本中自由配置 `dataset`、`threshold`（训练时门控优化的判定界限）、以及最重要的 `checkconf_list`（真实数据流模拟时测试的不同置信度阈值列表）。脚本将输出在目标机制下是否能仅利用早期序列就成功实现高精度指纹分类。
- **关联脚本**: `gate_main.py`。

### 📌 2.3 流式到达模拟评测（Stream Simulation）

为了更贴合实际环境，单独评测各主干+门控模型在逐个包或逐特征窗口到达场景下的“平均判定延迟”及“分类准确率”，请使用 `bash_stream_sim.sh`：

```bash
cd Prelude_main/Run
bash bash_stream_sim.sh
```

- **功能说明**: 脚本会读取已经训练好的机制断点检查文件（Checkpoints），并在模拟数据流逐块接收（Streaming）的状态下执行预测。支持传入多组不同的置信度判定阈值以形成关于“延迟 vs 准确率”的多粒度观测。
- **关联脚本**: `stream_sim.py`。

### 📌 2.4 其他实验与扩展分析

项目中同时提供了丰富的基础功能拓展与超参数敏感性（鲁棒性）实验验证脚本，命令规范明了：

- **以 `bash_sen_*.sh` 开头的脚本**（例如 `bash_sen_alpha.sh`, `bash_sen_Nwin.sh` 等）：用于对论文中的重点机制成分以及超参数进行消融与敏感度实验（比如探求不同数据增强维度、损失加权参数 $\alpha$ 和 $\beta$ ，或者预观测窗口容量设定的影响）。
- **以 `bash_ext_*.sh` 开头的脚本**：用于生成预测过程或结果统计的辅助量化指标（例如获取并导出不同分类阶段下的输出置信度分布特征等分析），为门控网络确定合适的判定阈值提供验证支撑。

您可以根据论文对应的评估章节去直接执行相关的 `.sh` 脚本。

## 3. 注意事项
1. **模型配置与模型构建**: 如需对模型学习率、Epochs 及针对某一特定结构参数的深度修改，请前往 `Prelude_main/Run/config/*.ini` 文件中进行适配调整。若是要新增网络模块组件，请修改 `Model/` 目录中的 Python 构造。
2. **检查点读写路径**: 模型相关的权值检查点将会统一存放至最外层的相对目录路径下（预设路径为 `../../checkpoints_test`，详见运行脚本顶部的路径变量 `checkpoint_path`），请保证运行权限或对应的上层存放目录容量可用。
3. **数据预挂载设定**: 数据集的读取接口封装由 `Run/utils_dataset_metric.py` 和 `Run/const.py` 管理。若在使用特定新设备时运行，需留意在 `const.py` 的相关数据基础路径下添加自身主机的专属加载路径映射。
